# Software Preservation

## 120-180 word message

I am sharing a small software-preservation project that may be of interest to people working on scientific code, computational history, or reproducibility. The project is a revival and empirical audit associated with Bombelli's 1987 causal-set simulated-annealing code/program. It provides a Python 3.12 port, small benchmark inputs, reproducible CPU runs, order-theoretic diagnostics, and documentation of the historical and computational context.

The aim is modest: to make an early computational physics program easier to inspect, run, cite, and discuss with modern tools. It does not claim to solve causal set theory, quantum gravity, or the causal set embedding problem. Instead, it treats the program as a citable software artifact and records how the revived implementation behaves under repeated runs and parameter choices.

Feedback on preservation practice, citation metadata, reproducibility, or historical framing would be welcome.
